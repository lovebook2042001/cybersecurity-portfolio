# Linux STIG-Style Security Audit

## Overview
This project demonstrates a Linux hardening and compliance-style security assessment inspired by STIG and RMF-oriented review methodologies. The assessment focused on reviewing system services, user privileges, SSH configurations, firewall settings, logging visibility, password policies, and security-related system configurations.

## Key Findings
The assessment identified system configurations related to SSH security, firewall visibility, running services, password policies, logging controls, and privilege management that impact Linux system hardening and attack surface exposure.
